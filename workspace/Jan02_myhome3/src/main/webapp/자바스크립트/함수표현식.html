함수 표현식이란? 함수인데 이름이 없는 함수를 뜻함
자바스크립트는 전역변수가 없기 때문에 함수가 처음부터 메모리를 차지하고 있음(C의 특징이기도 함, 자바랑은 많이 다름)
함수는 처음부터 메모리를 차지하고 있는데, 함수 표현식은 메모리를 차지하고 있지 않고 호출될 때 메모리가 할당됨
실행 뒤에는 메모리가 사라짐
보통은 이벤트핸들러나 콜백함수를 만들 때 주로 사용된다
가볍게 만들어서 사용하고 버리는 함수를 뜻한다

선언방법: 변수명=function(매개변수){ }
함수명이 따로 없고 왼쪽 변수에 함수의 주소를 전달하고 나중에 변수를 이용해 호출할 수 있다.
콜백함수는 함수를 만드는 사용자와 호출자가 다른 함수를 뜻한다.
함수를 만들어서 호출자에게 함수 주소를 전달해주면 호출자는 적당한 시점에 콜백함수를 호출한다
이벤트 핸들러를 구축하거나, 시스템의 일부분만을 별도의 코드로 작성해야 할 때
만드는 사람은 전체 시스템의 내부구조를 몰라도 정해진 범위 안에서 함수를 만들어 전달한다

동기식 처리 : A,B,C 세 개의 작업이 있을 때 A시작-종료, B시작-종료, C시작-종료가 순차적으로 죄다 일어남(속도 개느림)
비동기식 처리 : A를 시작하고 돌아가는 동안 바로 클라단에 return을 뱉음. 백단에서는 그동안 계속 작업을 진행시킴

자바는 비동기식 처리를 거의 하지 않음
